# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Running the tutor

```bash
# Activate the virtual environment first
source venv/Scripts/activate

# Run
python tutor.py
```

**Prerequisites before running:**
- Ollama desktop app must be open in the system tray
- Run `ollama pull llama3` once if the model isn't downloaded yet
- `ffmpeg.exe` and `ffprobe.exe` should be in the project root (auto-downloaded on first run if missing)

## Architecture: "Ear, Brain, Mouth"

The entire application is a single file (`tutor.py`) implementing a voice loop:

1. **Ear** — `SpeechRecognition` + `pyaudio` captures mic input to `input.wav`, then `faster-whisper` (`base.en`, CPU/INT8) transcribes it to text.
2. **Brain** — `ollama` sends the full conversation history to a local LLaMA 3 instance and gets a text response back.
3. **Mouth** — `RealtimeTTS` (Windows `SystemEngine` / SAPI) reads the response aloud.

The loop runs entirely offline with no API keys.

## Key configuration (top of tutor.py)

| Constant | Purpose |
|---|---|
| `MODEL_NAME` | Ollama model to use (default: `"llama3"`) |
| `WHISPER_SIZE` | Whisper model size: `"tiny.en"` (fast) → `"base.en"` → `"small.en"` (accurate) |
| `CV_CONTEXT` | Condensed student profile injected into both system prompts |

## Mode system

Two modes are selected at startup and each maps to a distinct system prompt:

- **Interview mode** (`INTERVIEW_SYSTEM_PROMPT`) — tutor asks one question at a time, gives English + content feedback, then waits. Advancement is gated by `is_next_trigger()`, which matches exact phrases (`"next"`, `"next question"`, `"go ahead"`, `"go on"`) to prevent accidental skips mid-answer.
- **Conversation mode** (`CONVERSATION_SYSTEM_PROMPT`) — free-form grammar-corrected chat.

In interview mode, the tutor fires the first question immediately by injecting `"Let's begin."` into the history before the user speaks.

## Session logging

Every session writes a timestamped transcript to `logs/session_<mode>_<YYYY-MM-DD_HH-MM-SS>.txt`. On "Goodbye", an end-of-session summary is generated by passing the full history back to Ollama with a summary prompt and appended to the log.
